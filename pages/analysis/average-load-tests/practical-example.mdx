# Praxisbeispiel

Die dargestellte Grafik des Praxisbeispiels (s.u.) offenbart eine auffällige Varianz in den
Antwortzeiten der virtuellen Threads. Diese Streuung erstreckt sich von minimalen Werten nahe
null bis hin zu maximalen Werten von bis zu 30.000 Millisekunden. Die deutliche Bandbreite in den
gemessenen Antwortzeiten weist darauf hin, dass die virtuellen Threads unterschiedlich auf die
gegebenen Anforderungen reagieren.

<ScalableImage
  src="/images/pb_response_time.png"
  alt="Messungen der Antwortzeit innerhalb des Praxisbeispiels (Platform Threads vs Virtual Threads)"
  description="Abbildung X - Quelle: Eigene Anfertigung"
/>

Wird die Anzahl der jeweils vorkommenden Dauer betrachtet so liegen die meisten Werte im Bereich
von unter 5000 Millisekunden. Es ist jedoch eine leichte Erhebung bei den normalen Threads im
Bereich über 500 Millisekunden zu erkennen, die näher betrachtet werden sollte.

<ScalableImage
  src="/images/pb_request_duration.png"
  alt="Messungen der Antwortzeit innerhalb des Praxisbeispiels als Balkendiagramm (Platform Threads vs Virtual Threads)"
  description="Abbildung X - Quelle: Eigene Anfertigung"
/>

Werden die Antwortzeiten über 500 Millisekunden betrachtet, wird sichtbar, dass die normalen
Threads dort noch einige male vorkommen. Was die im Durchschnitt bessere Performance der
Virtuellen Threads erklärt.
