## Einführung

Soll eine Spring Boot Anwendung reaktive entwickelt werden, so muss der gesamte Stack reaktiv sein.
Somit müssen auch die Abhängigkeiten in der `pom.xml` angepasst werden. Anschließend müssen
entsprechende Anpassungen an bestehenden Anwendungen vorgenommen werden und neue Anwendungen
verwenden andere Interfaces und müssen andere Konfigurationen vornehmen.

Im Folgenden wird exemplarisch gezeigt, welche Abhängigkeiten für eine reaktive Spring Boot Anwendung
verwendet werden können. Darüber hinaus wird ausschnittweise gezeigt, wie eine RESTful Microservice Architektur mit
Datenbankanbindung und Kommunikation mit RabbitMQ reaktiv verwendet werden können.
